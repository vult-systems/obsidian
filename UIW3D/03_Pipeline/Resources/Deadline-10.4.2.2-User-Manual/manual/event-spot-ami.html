Overview¶

As part of creating Spot Fleet Requests, you will require an Amazon Machine Image (AMI) for each Spot Fleet you wish to start.

The AMI’s represent the base state that each of you instances will start with.

Recommended Steps¶

We recommend the following process when creating an AMI:

  • Ensure the OS (operating system) is using UTC time.

  • Install the Deadline Client.

  • Install your render application of choice.

  • Configure your file system to automatically mount on start.

  • Configure Deadline to start automatically.

  • Configure Deadline connectivity.

See also

To see an example of creating an AMI for the Spot Event Plugin, please take a look at our Studio in the Cloud Tutorial.

Note

It is recommended that you do NOT install your Deadline Client or Render Software centrally on your Linux render nodes.
Centrally installing your software can cause slowdowns when starting Deadline or your renders.
