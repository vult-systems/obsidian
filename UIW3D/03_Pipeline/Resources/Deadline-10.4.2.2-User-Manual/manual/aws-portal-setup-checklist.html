If you’re familiar with Deadline 10 and looking to start using AWS Portal, we recommend starting here.

Note

While AWS Portal can enable cloud rendering in Deadline 10, AWS Deadline Cloud is a newer offering that has been built specifically for the cloud. It is a fully managed service that does not require installation or maintenance of infrastructure (e.g., repository, database, or license server). Worker fleet auto-scaling, asset synching, and licensing are all managed natively within AWS by Deadline Cloud. See here for more information on Deadline Cloud and its capabilities.

AWS Portal Setup Checklist¶

AWS Account Setup¶

  • Create AWS account

  • Create IAM Admin user using the AWS console

  • Create IAM AWSPortal user using the AWS console

  • Request AWS account limit increases if necessary

Software Installation¶

  • Install the Deadline Client

  • Install AWS Portal Asset Server and AWS Portal Link (single installer)

  • Start a Remote Connection Server (RCS)

Configuration¶

  • Specify your Remote Connection Server

  • Specify your Asset Server’s Local IP Address

  • Configure asset root directories in the Asset Server Settings dialog (if not done when installing the Asset Server)

Usage-Based Licensing¶

Warning

Topics covered in this page refer to legacy Thinkbox usage-based licensing (UBL), which is no longer available as of September 30, 2025. The UBL feature in Deadline 10 is now available through AWS Deadline Cloud UBL. For detailed setup instructions, please see the Deadline Cloud UBL Documentation.

For more details, please refer to the marketplace website.

  • Create a Thinkbox Marketplace account

  • Buy credits through the Thinkbox Marketplace

  • Configure your Cloud License Server

  • Download your UBL certificates and set your UBL sync directory

  • Set UBL Deadline Limits (After starting an infrastructure)

This step should only be completed if you are using Bring Your Own License

  • Set up Bring Your Own License

  • Start an Infrastructure
