Render Output Not Coming Back¶

What Went Wrong¶

Output is not coming back from renders.

Figuring Out What Went Wrong¶

Check the Job Report¶

First check the job report to make sure it’s not an issue with the scene.

Check CloudWatch Logs¶

Go to CloudWatch

  1. Open the AWS Management Console.

  2. Go to Services > CloudWatch > Logs

Check the Central Asset Server Log

  1. Click the /thinkbox/S3BackedCache/central Log Group.

  2. The newest log will be at the top, click it.

  3. Check the log for any errors.

Check the Worker Asset Log

  1. Click the /thinkbox/S3BackedCache/worker Log Group.

  2. In the Filter field input the stack name.

  3. Select the Worker log of the machine not returning the output. It will have the instance id appended to the end of it.

Check the AWS Asset Server Log

Click here for how to find the AWS Portal Asset Server logs

Solutions¶

Check Output Paths in Job Report¶

Check the Job Report to make sure the output path is correct. Also make sure that if the output path is correct, it is also in one of your AWS Asset Server Root Directories.

Check File Roots¶

Check to make sure that your output path is in one of the AWS Asset Server Root Directories. You can check this by:

  1. Opening the Deadline Monitor.

  2. In Power User mode open the AWS Portal panel and login if necessary.

  3. Go to Tools > Configure Asset Server.

  4. Check to make sure that the output path is in the Root Directories field.

  5. If it isn’t either add it or change the output path to be in one of the Root Directories.

Check Mapped Drives¶

Ensure that your Mapped Drives are configured correctly. More reading on Mapped Drives here
